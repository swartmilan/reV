reV.handlers.cli_collect module¶

reV-collect¶

reV Collect Command Line Interface

reV-collect [OPTIONS] COMMAND [ARGS]...

Options

-n, --name <name>¶

Collection job name. Default is “reV_collect”.

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

direct¶

Main entry point for collection with context passing.

reV-collect direct [OPTIONS] COMMAND [ARGS]...

Options

-f, --h5_file <h5_file>¶

Required H5 file to be collected into.

-d, --h5_dir <h5_dir>¶

Required Directory containing h5 files to collect.

-pp, --project_points <project_points>¶

Project points file representing the full collection scope.

-ds, --dsets <dsets>¶

Required Dataset names to be collected.

-fp, --file_prefix <file_prefix>¶

File prefix found in the h5 file names to be collected.

-ld, --log_dir <log_dir>¶

Directory to put log files.

-p, --purge_chunks¶

Flag to delete chunked files after collection.

-v, --verbose¶

Flag to turn on debug logging.

collect¶

Run collection on local worker.

reV-collect direct collect [OPTIONS]

Options

-v, --verbose¶

Flag to turn on debug logging.

collect-slurm¶

Run collection on HPC via SLURM job submission.

reV-collect direct collect-slurm [OPTIONS]

Options

-a, --alloc <alloc>¶

SLURM allocation account name. Default is “rev”.

-mem, --memory <memory>¶

SLURM node memory request in GB. Default is None

-wt, --walltime <walltime>¶

SLURM walltime request in hours. Default is 1.0

-l, --feature <feature>¶

Additional flags for SLURM job. Format is “–qos=high” or “–depend=[state:job_id]”. Default is None.

-env, --conda_env <conda_env>¶

Conda env to activate

-mod, --module <module>¶

Module to load

-sout, --stdout_path <stdout_path>¶

Subprocess standard output path. Default is ./out/stdout

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

from-config¶

Run reV gen from a config file.

reV-collect from-config [OPTIONS]

Options

-c, --config_file <config_file>¶

Required reV collection configuration json file.

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

valid-config-keys¶

Echo the valid Collect config keys

reV-collect valid-config-keys [OPTIONS]

File collection CLI entry points.

reV.handlers.cli_collect.get_node_cmd(name, h5_file, h5_dir, project_points, dsets, file_prefix=None, log_dir='./logs/', purge_chunks=False, verbose=False)[source]¶

Make a reV collection local CLI call string.

Parameters

  • name (str) – reV collection jobname.

  • h5_file (str) – Path to .h5 file into which data will be collected

  • h5_dir (str) – Root directory containing .h5 files to combine

  • project_points (str | slice | list | pandas.DataFrame) – Project points that correspond to the full collection of points contained in the .h5 files to be collected

  • dsets (list) – List of datasets (strings) to be collected.

  • file_prefix (str) – .h5 file prefix, if None collect all files on h5_dir

  • log_dir (str) – Log directory.

  • purge_chunks (bool) – Flag to delete the chunked files after collection.

  • verbose (bool) – Flag to turn on DEBUG logging

Returns

cmd (str) – Single line command line argument to call the following CLI with appropriately formatted arguments based on input args:

python -m reV.handlers.cli_collect [args] collect
