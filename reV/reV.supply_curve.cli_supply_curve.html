reV.supply_curve.cli_supply_curve module¶

reV-supply-curve¶

reV Supply Curve Command Line Interface

reV-supply-curve [OPTIONS] COMMAND [ARGS]...

Options

-n, --name <name>¶

Job name. Default is “reV-sc”.

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

direct¶

reV Supply Curve CLI.

reV-supply-curve direct [OPTIONS] COMMAND [ARGS]...

Options

-sc, --sc_points <sc_points>¶

Required Supply curve point summary table (.csv or .json).

-tt, --trans_table <trans_table>¶

Required Supply curve transmission mapping table (.csv or .json).

-fcr, --fixed_charge_rate <fixed_charge_rate>¶

Required Fixed charge rate used to compute LCOT

-scf, --sc_features <sc_features>¶

Table containing additional supply curve features (.csv or .json)

-tc, --transmission_costs <transmission_costs>¶

Table or serialized dict of transmission cost inputs.

-so, --sort_on <sort_on>¶

The supply curve table column label to sort on. This determines the ordering of the SC buildout algorithm.

-ott, --offshore_trans_table <offshore_trans_table>¶

Path to offshore transmission table, if None offshore sc points will not be included, by default None

-wd, --wind_dirs <wind_dirs>¶

Path to .csv containing reVX.wind_dirs.wind_dirs.WindDirs output with the neighboring supply curve point gids and power-rose value at each cardinal direction

-dirs, --n_dirs <n_dirs>¶

Number of prominent directions to use

-dw, --downwind¶

Flag to remove downwind neighbors as well as upwind neighbors

-oc, --offshore_compete¶

Flag as to whether offshore farms should be included during CompetitiveWindFarms, by default False

-mw, --max_workers <max_workers>¶

Number of workers to use to compute lcot, if > 1 run in parallel. None uses all available cpus.

-o, --out_dir <out_dir>¶

Directory to save aggregation summary output.

-ld, --log_dir <log_dir>¶

Directory to save aggregation logs.

-s, --simple¶

Flag to turn on simple supply curve calculation.

-ll, --line_limited¶

Flag to turn on line-limited substation capacity calculation (legacy methodology). Alternative is multi-line spread capacity.

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

slurm¶

slurm (eagle) submission tool for reV supply curve.

reV-supply-curve direct slurm [OPTIONS]

Options

-a, --alloc <alloc>¶

Required SLURM allocation account name.

-mem, --memory <memory>¶

SLURM node memory request in GB. Default is None

-wt, --walltime <walltime>¶

SLURM walltime request in hours. Default is 1.0

-l, --feature <feature>¶

Additional flags for SLURM job. Format is “–qos=high” or “–depend=[state:job_id]”. Default is None.

-mod, --module <module>¶

Module to load

-env, --conda_env <conda_env>¶

Conda env to activate

-sout, --stdout_path <stdout_path>¶

Subprocess standard output path. Default is in out_dir.

from-config¶

Run reV supply curve compute from a config file.

reV-supply-curve from-config [OPTIONS]

Options

-c, --config_file <config_file>¶

Required reV supply curve configuration json file.

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

valid-config-keys¶

Echo the valid SupplyCurve config keys

reV-supply-curve valid-config-keys [OPTIONS]
