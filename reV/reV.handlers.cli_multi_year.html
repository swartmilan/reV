reV.handlers.cli_multi_year module¶

reV-multiyear¶

reV Multi-Year Command Line Interface

reV-multiyear [OPTIONS] COMMAND [ARGS]...

Options

-n, --name <name>¶

Multi-year job name. Default is “reV_multi-year”.

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

direct¶

Main entry point for collection with context passing.

reV-multiyear direct [OPTIONS] COMMAND [ARGS]...

Options

-f, --my_file <my_file>¶

Required h5 file to use for multi-year collection.

-v, --verbose¶

Flag to turn on debug logging.

multi-year¶

Run multi year collection and means on local worker.

reV-multiyear direct multi-year [OPTIONS]

Options

-g, --group <group>¶

Group to collect into. Useful for collecting multiple scenarios into a single file.

-sf, --source_files <source_files>¶

Required List of files to collect from.

-ds, --dsets <dsets>¶

Required Dataset names to be collected. If means, multi-year means will be computed.

-v, --verbose¶

Flag to turn on debug logging.

multi-year-groups¶

Run multi year collection and means for multiple groups.

reV-multiyear direct multi-year-groups [OPTIONS]

Options

-gp, --group_params <group_params>¶

Required List of groups and their parameters(group, source_files, dsets) to collect

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

multi-year-slurm¶

Run multi year collection and means on HPC via SLURM job submission.

reV-multiyear direct multi-year-slurm [OPTIONS]

Options

-a, --alloc <alloc>¶

SLURM allocation account name. Default is “rev”.

-wt, --walltime <walltime>¶

SLURM walltime request in hours. Default is 1.0

-l, --feature <feature>¶

Additional flags for SLURM job. Format is “–qos=high” or “–depend=[state:job_id]”. Default is None.

-mem, --memory <memory>¶

SLURM node memory request in GB. Default is None

-env, --conda_env <conda_env>¶

Conda env to activate

-mod, --module <module>¶

Module to load

-sout, --stdout_path <stdout_path>¶

Subprocess standard output path. Default is ./out/stdout

-gp, --group_params <group_params>¶

Required List of groups and their parameters(group, source_files, dsets) to collect

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

from-config¶

Run reV gen from a config file.

reV-multiyear from-config [OPTIONS]

Options

-c, --config_file <config_file>¶

Required reV multi-year configuration json file.

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

valid-config-keys¶

Echo the valid MultiYear config keys

reV-multiyear valid-config-keys [OPTIONS]

Multi-year means CLI entry points.

reV.handlers.cli_multi_year.get_slurm_cmd(name, my_file, group_params, verbose=False)[source]¶

Make a reV multi-year collection local CLI call string.

Parameters

  • name (str) – reV collection jobname.

  • my_file (str) – Path to .h5 file to use for multi-year collection.

  • group_params (list) – List of groups and their parameters to collect

  • verbose (bool) – Flag to turn on DEBUG logging

Returns

cmd (str) – Argument to call the neccesary CLI calls on the node to collect desired groups
