reV.rep_profiles.cli_rep_profiles module¶

reV-rep-profiles¶

reV Representative Profiles Command Line Interface

reV-rep-profiles [OPTIONS] COMMAND [ARGS]...

Options

-n, --name <name>¶

Job name. Default is “reV-rep_profiles”.

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

direct¶

reV representative profiles CLI.

reV-rep-profiles direct [OPTIONS] COMMAND [ARGS]...

Options

-g, --gen_fpath <gen_fpath>¶

Required Filepath to reV gen file.

-r, --rev_summary <rev_summary>¶

Required Filepath to reV SC summary (agg) file.

-rc, --reg_cols <reg_cols>¶

List of column rev summary column labels to define regions to get rep profiles for.

-cf, --cf_dset <cf_dset>¶

Capacity factor dataset in gen_fpath to get profiles from

-rm, --rep_method <rep_method>¶

String identifier for representative method (e.g. meanoid, medianoid).

-em, --err_method <err_method>¶

String identifier for error method (e.g. rmse, mae, mbe).

-w, --weight <weight>¶

The supply curve column to use for a weighted average in the representative profile meanoid algorithm. Default weighting factor is “gid_counts”.

-np, --n_profiles <n_profiles>¶

Number of representative profiles to save.

-od, --out_dir <out_dir>¶

Directory to save rep profile output h5.

-ld, --log_dir <log_dir>¶

Directory to save rep profile logs.

-mw, --max_workers <max_workers>¶

Number of parallel workers. 1 will run in serial. None will use all available.

-agg, --aggregate_profiles¶

Flag to calculate the aggregate (weighted meanoid) profile for each supply curve point. This behavior is instead of finding the single profile per region closest to the meanoid.

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

slurm¶

slurm (Eagle) submission tool for reV representative profiles.

reV-rep-profiles direct slurm [OPTIONS]

Options

-a, --alloc <alloc>¶

Required SLURM allocation account name.

-mem, --memory <memory>¶

SLURM node memory request in GB. Default is None

-wt, --walltime <walltime>¶

SLURM walltime request in hours for single year rep_profiles run. Default is 1.0

-l, --feature <feature>¶

Additional flags for SLURM job. Format is “–qos=high” or “–depend=[state:job_id]”. Default is None.

-env, --conda_env <conda_env>¶

Conda env to activate

-mod, --module <module>¶

Module to load

-sout, --stdout_path <stdout_path>¶

Subprocess standard output path. Default is in out_dir.

from-config¶

Run reV representative profiles from a config file.

reV-rep-profiles from-config [OPTIONS]

Options

-c, --config_file <config_file>¶

Required reV representative profiles configuration json file.

-v, --verbose¶

Flag to turn on debug logging. Default is not verbose.

valid-config-keys¶

Echo the valid RepProfiles config keys

reV-rep-profiles valid-config-keys [OPTIONS]
