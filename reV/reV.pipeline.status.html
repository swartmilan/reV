reV.pipeline.status module¶

reV job status manager.

class reV.pipeline.status.Status(status_dir, name=None, hardware='eagle', subprocess_manager=None)[source]¶

Bases: dict

Base class for reV data pipeline health and status information.

Parameters

  • status_dir (str) – Directory to with json status file.

  • name (str | None) – Optional job name for status. Will look for the file “{name}_status.json” in the status_dir.

  • hardware (str) – Name of hardware that this pipeline is being run on: eagle, local. Defaults to “eagle”. This specifies how job are queried for status.

  • subprocess_manager (None | SLURM) – Optional initialized subprocess manager to use to check job statuses. This can be input with cached queue data to avoid constantly querying the HPC.
